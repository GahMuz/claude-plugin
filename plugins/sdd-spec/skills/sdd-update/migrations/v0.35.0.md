# v0.35.0

## Changements

- Nouveau skill `/analyse` — analyse multi-dimensionnelle (qualité, architecture, conformité RGPD/DORA)
- Suppression de `/doc analyse` — sous-commande retirée de `/doc`
- Suppression de l'agent `doc-analyser` — remplacé par 3 agents spécialisés

## Migration `.sdd/config.json`

Remplacer la clé `doc-analyser` par les trois nouvelles clés d'agents :

```json
// Avant
"models": {
  ...
  "doc-analyser": "sonnet"
}

// Après
"models": {
  ...
  "analyse-quality": "sonnet",
  "analyse-architecture": "opus",
  "analyse-compliance": "sonnet"
}
```

## Répertoire de sortie

Les analyses sont désormais dans `.sdd/analyses/<module>/` (plus dans `.sdd/docs/modules/<module>/`).
Les fichiers `analyse-<module>.md`, `improvement-<module>.md`, `missing-rules-<module>.md` dans `.sdd/docs/` peuvent être supprimés — ils ne sont plus générés.
