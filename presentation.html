TDD structurel — pas optionnel

Chaque tâche code → décomposée automatiquement :

RED   Tests en échec dérivés des critères d'acceptation REQ
GREEN   Code minimal pour faire passer les tests
REFACTOR   Nettoyage (optionnel si déjà propre)

Anti-patterns détectés :

  • Tester le mock plutôt que le code réel
  • Méthodes reset() test-only dans les classes prod
  • Déclarer "tests pass" sans les avoir exécutés

Debugging systématique

Règle de fer : jamais de correctif sans cause racine identifiée.

Phase 1  Investigation — lire l'erreur, tracer la call chain
Phase 2  Pattern analysis — comparer avec un cas qui fonctionne
Phase 3  Hypothèse unique — un changement minimal à la fois
Phase 4  Fix & Verify — cause, pas symptôme + suite complète
Seuil d'escalade
Après 3 tentatives échouées : stop.
La tâche est marquée [!], l'orchestrateur dispatche spec-deep-dive.
Defense in depth
Après la fix : validation ajoutée aux 4 couches pour rendre le bug structurellement impossible.
