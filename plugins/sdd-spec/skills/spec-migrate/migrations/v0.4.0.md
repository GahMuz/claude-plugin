# v0.4.0 — Migration .specs/ → .sdd/

Restructuration du répertoire de données : séparation des specs et de la documentation générée, introduction du registry et du datage des specs.

## Prérequis

```bash
# Vérifier que l'ancienne structure existe
[ -d ".specs" ] || { echo "Rien à migrer : .specs/ absent."; exit 0; }
# Vérifier que la nouvelle n'existe pas déjà
[ -d ".sdd" ] && { echo "Erreur : .sdd/ existe déjà. Migration déjà effectuée ?"; exit 1; }
```

## Step 1 : Créer la nouvelle structure

```bash
mkdir -p .sdd/specs .sdd/docs
```

## Step 2 : Migrer la configuration

```bash
cp .specs/config.json .sdd/config.json
```

Ajouter le champ `schemaVersion` dans `.sdd/config.json` (via Edit) :
```json
"schemaVersion": "0.4.0"
```

## Step 3 : Migrer la documentation générée

Si `.specs/doc/` existe :
```bash
cp -r .specs/doc/. .sdd/docs/
```

## Step 4 : Migrer chaque spec

Pour chaque répertoire dans `.specs/` (exclure `doc/`) :

1. Lire `createdAt` dans `.specs/<spec-id>/state.json`
   - Si absent : lire la première date dans `.specs/<spec-id>/log.md`
   - Si absent : utiliser `git log -1 --format=%ai -- .specs/<spec-id>/state.json`
   - Fallback : date du jour
2. Extraire `YYYY/MM` depuis la date
3. Créer le répertoire daté et copier :
   ```bash
   mkdir -p ".sdd/specs/YYYY/MM/<spec-id>"
   cp -r ".specs/<spec-id>/." ".sdd/specs/YYYY/MM/<spec-id>/"
   ```

## Step 5 : Créer le registry

Lire `title` et `currentPhase` dans chaque `state.json` migré.
Écrire `.sdd/specs/registry.md` :

```markdown
# Registre des specs

| Identifiant | Titre | Période | Statut | Requirement | Design | Plan |
|-------------|-------|---------|--------|-------------|--------|------|
| <spec-id> | <title> | YYYY/MM | <currentPhase> | [REQ](YYYY/MM/<spec-id>/requirement.md) | [DES](YYYY/MM/<spec-id>/design.md) | [PLAN](YYYY/MM/<spec-id>/plan.md) |
```

## Step 6 : Vérification

Pour chaque spec migrée, vérifier via Glob que `state.json` existe dans la destination.
Si une vérification échoue : signaler l'erreur et **ne pas supprimer `.specs/`**.

## Step 7 : Nettoyage

Si toutes les vérifications passent :
"Migration vérifiée. Supprimer l'ancienne structure `.specs/` ? (oui / non)"

Si oui :
```bash
rm -rf .specs/
```
