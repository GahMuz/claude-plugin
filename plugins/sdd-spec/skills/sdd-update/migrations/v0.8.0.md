# v0.8.0 — Suppression du champ richRequirements

Le champ `richRequirements` est retiré de `.sdd/config.json` (il était toujours `false` par défaut et n'influençait aucun comportement).

## Step 1 : Vérifier la présence du champ

```bash
grep -q '"richRequirements"' .sdd/config.json \
  && echo "Migration nécessaire." \
  || { echo "Champ absent — rien à faire."; exit 0; }
```

## Step 2 : Supprimer le champ

Ouvrir `.sdd/config.json` via Edit et retirer la ligne :

```json
"richRequirements": false,
```

(ou `true` si l'utilisateur avait répondu oui lors du `/spec-init`).

## Step 3 : Mettre à jour schemaVersion

Dans `.sdd/config.json`, remplacer `"schemaVersion": "0.7.0"` par `"schemaVersion": "0.8.0"`.
