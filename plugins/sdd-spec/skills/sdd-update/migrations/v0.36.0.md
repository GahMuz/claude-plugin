# v0.36.0 — Refactoring des analyses (/doc analyse → /analyse)

La sous-commande `/doc analyse` est supprimée. Les analyses multi-dimensionnelles
sont désormais gérées par le skill `/analyse` dans un répertoire dédié `.sdd/analyses/`.

## 1. Mettre à jour `.sdd/config.json` — clé `models.doc-analyser`

Remplacer la clé `doc-analyser` par trois clés distinctes :

```bash
# Vérifier la valeur actuelle
cat .sdd/config.json | grep doc-analyser
```

Dans `.sdd/config.json`, remplacer :

```json
"doc-analyser": "sonnet"
```

par :

```json
"analyse-quality": "sonnet",
"analyse-architecture": "opus",
"analyse-compliance": "sonnet"
```

> Ajustez les modèles selon vos préférences — ce sont les valeurs par défaut.

## 2. Créer le répertoire `.sdd/analyses/`

```bash
mkdir -p .sdd/analyses
```

## 3. Nettoyer les anciens fichiers d'analyse (si présents)

Les fichiers `analyse-<module>.md`, `improvement-<module>.md` et
`missing-rules-<module>.md` dans `.sdd/docs/modules/<module>/` sont obsolètes.
Vérifier leur présence :

```bash
find .sdd/docs/modules -name "analyse-*.md" -o -name "improvement-*.md" -o -name "missing-rules-*.md"
```

Si des fichiers sont trouvés, ils peuvent être supprimés (leur contenu n'est plus
exploité par le workflow). Les nouvelles analyses sont générées via `/analyse <module>`
dans `.sdd/analyses/<module>/`.
