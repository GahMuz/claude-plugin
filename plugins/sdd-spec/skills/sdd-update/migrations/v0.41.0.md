# v0.41.0 — Ajout du support sdd-graph dans config.json

Deux nouveaux champs sont requis dans `.sdd/config.json` pour la compatibilité avec le plugin `sdd-graph`.

## 1. Ajouter les clés `models.graph-builder` et `models.graph-query`

Vérifier si les clés sont déjà présentes :

```bash
grep -E "graph-builder|graph-query" .sdd/config.json
```

Si absentes, dans `.sdd/config.json`, remplacer la dernière entrée du bloc `models` :

```json
"analyse-compliance": "sonnet"
```

par :

```json
"analyse-compliance": "sonnet",
"graph-builder": "haiku",
"graph-query": "haiku"
```

## 2. Ajouter le bloc `graph`

Dans `.sdd/config.json`, ajouter après le bloc `models` (avant `"createdAt"`) :

```json
"graph": {
  "enabled": false,
  "stacks": [],
  "sourcePaths": {},
  "stalenessThresholdDays": 7
},
```

> `graph.enabled` reste `false` jusqu'à l'installation et la configuration du plugin `sdd-graph`.
> Pour activer le graphe sur un projet Java, relancez `/sdd-init` ou éditez manuellement `graph.enabled`, `stacks` et `sourcePaths`.
